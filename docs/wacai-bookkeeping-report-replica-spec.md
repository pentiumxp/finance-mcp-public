# 挖财记账与统计复刻规格

日期：2026-05-29

## 1. 目标

Finance 的记账与统计页面按挖财的业务流程、信息层级、字段顺序、颜色、布局、图标语义和交互动效做高保真复刻。这里的“复刻”指本地 Finance Web/PWA 的用户体验和视觉结构尽量与挖财一致，不再采用 Finance 自己的 UI 风格；但不改变本项目的架构边界：

- Finance 仍是独立本地财务事实系统。
- Hermes Mobile 只通过 `finance` MCP toolset 使用 Finance。
- Web UI 是本地管理和验证界面，不嵌入 Hermes Mobile。
- UI 视觉语言以挖财为准；仅保留 Hermes Mobile 的硬性交互规则：固定页眉、禁止手动缩放、二级页左上角返回、左缘右滑返回、无单独退出按钮。

## 1.1 视觉复刻原则

本项目的 Finance PWA 视觉目标调整为：

- 页面布局、颜色、圆角、间距、字号、按钮位置、tab 样式、卡片层级、底部导航和键盘样式以挖财为第一参照。
- 不再使用 Finance 自己的“灰蓝控制台”视觉语言作为默认外观。
- Hermes Mobile 规则只约束导航和页面行为，不约束 Finance 的颜色和控件外观。
- 页面默认支持深色模式优先；浅色模式仍要可用，但高保真目标优先对齐用户提供的挖财深色截图。
- 图标需要本地化重绘成 SVG/CSS icon 或项目内静态资产；不直接热链、抓取或依赖挖财 APK 内私有资源。
- 背景图需要项目内位图资产；可以用等效生成/重绘背景替代，不使用纯色或普通渐变糊弄。
- 所有中文文案必须保持正常 UTF-8 显示，乱码视为阻断问题。

## 2. 复刻依据

当前依据来自：

- 用户上传的挖财记账截图。
- Android 模拟器中挖财页面采样。
- 2025 年挖财 XLSX 导出数据。
- 当前 Finance 代码和数据库能力。

不得把截图理解为“只做外观”。挖财的核心闭环是：

```text
分类优先记账 -> 金额键盘 -> 标签/商家/账户/成员 -> 账单明细 -> 统计报表 -> 下钻明细
```

## 3. 记账页复刻规格

### 3.1 页面结构

记账页为全屏页面，不显示底部导航。页面从上到下分为：

1. 状态区和账本区。
2. 类型 tabs。
3. 当前分类与金额卡片。
4. 分类图标网格。
5. 备注输入。
6. 日期、账户、成员、商家、标签、附件等横向 chips。
7. 固定底部计算键盘。

### 3.1.1 记账页颜色和尺寸基线

深色高保真目标：

| 区域 | 规格 |
|---|---|
| 页面背景 | `#000000` 到 `#050505`，主体内容黑底 |
| 顶部背景 | 模糊蓝色图片背景，叠加冷蓝遮罩 |
| 当前分类卡片 | 深色截图使用 `#1f1f22`，浅色截图使用 `#ffffff` |
| 卡片圆角 | 18px 到 22px |
| 挖财绿色 | `#28cdb8` 附近，用于选中分类圆和报表主图标 |
| 保存橙色 | `#ff7048` 到 `#ff744c` |
| 键盘背景 | `#4d515f` 附近 |
| 键盘分割线 | `rgba(255,255,255,0.06)` |
| 普通图标圆底 | 深色：`#161719`；浅色：`#ececef` |
| 普通图标 | 深色：`#4f535c`；浅色：`#9ca3af` |
| 主文字 | 深色：`#f5f5f5`；浅色：`#24262c` |
| 次文字 | 深色：`#9da0aa`；浅色：`#8d94a3` |

尺寸基线以 390px-430px 宽手机为基准：

- 顶部背景高度：约 280px。
- 类型 tab 行高度：52px。
- 当前分类卡片左右边距：24px 到 28px。
- 当前分类卡片高度：约 96px 到 112px。
- 分类图标网格：4 列，行距 22px 到 28px。
- 分类图标圆：48px 到 56px。
- 分类文字字号：16px 到 18px。
- 金额字号：40px 到 46px。
- 底部 chips 高度：38px 到 44px。
- 键盘总高度：约 254px，加安全区。
- 键盘数字字号：26px 到 30px。

### 3.2 顶部区域

必须包含：

- 左上角返回按钮。
- 中间账本名称，例如“日常账本”，右侧带下拉箭头。
- 右侧收藏/常用图标。
- 右侧设置图标。
- 背景使用挖财式模糊图片或等效位图背景，不使用纯渐变替代。

类型 tabs：

- `收入`
- `支出`
- `转账`
- `借贷`

状态：

- 默认选中 `支出`。
- 选中项为白色文字加短下划线。
- 未选中项为白色半透明文字。

### 3.3 当前分类与金额卡片

卡片位于顶部背景下沿，圆角大卡片样式。

内容：

- 左侧为当前分类图标，圆形背景。
- 中间为当前分类名称。
- 右侧为金额输入结果。

默认：

- 支出默认分类按挖财使用频率优先，首选 `午餐`。
- 金额默认显示 `0.00`。

交互：

- 点击分类网格中的分类后，当前分类卡片同步更新。
- 金额键盘输入实时更新金额。
- 切换收入/支出/转账/借贷后，分类网格和当前分类重置到该类型默认分类。

### 3.4 分类图标网格

分类是记账页第一优先级。分类网格必须在金额扩展字段前展示。

布局：

- 4 列网格。
- 每个分类项由圆形图标和文本组成。
- 图标圆直径约 50px。
- 未选中图标背景为深色/浅色模式下的中性圆底。
- 选中图标背景为挖财绿色，图标为白色。
- 文本最多两行，不能遮挡或截断成乱码。

当前截图出现的支出快捷分类必须覆盖：

- 早餐
- 午餐
- 晚餐
- 医疗药品
- 电子数码
- 养生保健
- 生活用品
- 正版软件
- 大家电
- 服饰鞋包
- 酒
- 牛奶
- 水
- 健康有机
- 电脑宽带
- 新风

后续从 2025 XLSX 中出现但快捷区未显示的分类仍要进入完整分类列表，可通过“更多/展开”进入。

### 3.5 图标系统

不能再用分类名称首字作为图标。必须为分类建立稳定的 `icon_key`。

数据模型目标：

```text
finance_categories.icon_key
finance_categories.icon_color
```

前端渲染目标：

```html
<span class="finance-category-icon icon-food-lunch"></span>
```

图标来源：

- 优先使用项目内 SVG mask/CSS icon。
- 不依赖远程图标 CDN。
- 不把截图裁剪图标直接作为唯一来源，避免模糊和授权问题。

### 3.6 备注和元信息 chips

备注：

- 占一行，placeholder 为“输入备注...”。
- 位于分类网格下方、键盘上方。

chips 顺序必须与挖财一致：

1. 日期时间，例如 `今天 20:31`。
2. 账户，例如 `现金`。
3. 成员，例如 `自己`。
4. 标签。
5. 商家。
6. 附件/拍照按钮。

字段要求：

- 日期点击后进入日期时间选择。
- 账户点击后进入账户选择。
- 成员点击后进入成员选择。
- 标签点击后进入标签选择。
- 商家点击后进入商家输入/选择。
- 附件点击后进入收据/发票附件入口。

当前第一版可以先保留 select/input 实现，但视觉必须是 chip，不得显示原生表单块。

### 3.7 金额键盘

键盘固定在底部，不能随页面内容滚走。

布局为 5 列 4 行：

```text
1  2  3  +  删除
4  5  6  -  再记
7  8  9  ×÷ 保存
   0  .  =
```

要求：

- 键盘背景为深灰。
- 普通键白字。
- `保存` 为橙色大按钮，占右下两行。
- 删除使用退格图标，不使用乱码字符。
- `×÷` 是一个复合运算键，行为上可以先输入 `*` 或 `/`，但显示必须和挖财一致。
- 支持 `+ - × ÷ =` 简单计算。
- 提交前计算表达式并归一化金额。

### 3.8 保存和再记

保存：

- 校验金额必须大于 0。
- 校验分类、账户、成员。
- 写入交易、更新账户余额、写审计。
- 成功后返回首页或保留在记账页，取决于入口模式。

再记：

- 提交当前交易。
- 清空金额和备注。
- 保持当前分类、账户、成员、日期默认推进到当前时间。
- 留在记账页继续下一笔。

## 4. 统计页复刻规格

### 4.1 页面结构

统计页从上到下分为：

1. 状态栏/顶部筛选。
2. 周期 segmented control。
3. 日期范围行。
4. 维度横向 tabs。
5. 报表卡片。
6. 底部导航。

### 4.1.1 统计页颜色和尺寸基线

深色高保真目标：

| 区域 | 规格 |
|---|---|
| 页面背景 | `#000000` |
| 顶部筛选栏背景 | `#1e1f22` 到 `#2d2f33` |
| segmented 背景 | `#303235` |
| segmented 选中项背景 | `#222326` |
| 选中橙色 | `#ff7048` |
| 报表卡片 | `#1d1d20` |
| 卡片圆角 | 18px 到 22px |
| 主文字 | `#f2f2f4` |
| 次文字 | `#9a9ca3` |
| 分割线 | `rgba(255,255,255,0.07)` |
| 底部导航背景 | `#1b1b1d` |
| 底部导航选中 | 橙色文字和图标 |

尺寸基线：

- 顶部筛选高度：约 148px 到 180px，包含周期和日期范围。
- 周期 segmented 左右边距：约 52px。
- 维度 tab 行高度：72px。
- 维度 tab 字号：22px 到 26px。
- 选中下划线宽度：28px 到 36px，高度 4px。
- 报表卡片边距：16px。
- 报表卡片内边距：22px 到 26px。
- donut 外径：约 220px 到 260px。
- 排行行高度：110px 到 132px。
- 排行图标圆：54px 到 64px。
- 金额字号：24px 到 30px。
- 笔数/百分比字号：16px 到 20px。

### 4.2 周期筛选

顶部 segmented control：

- `全部`
- `年`
- `月`
- `更多`

要求：

- 当前选中项为橙色文字。
- 当前选中项有深色圆角背景。
- 其他项有分隔线。
- `更多` 右侧带下拉箭头。

日期范围行：

- 形如 `2005/12/31 ~ 2026/11/25`。
- 开始和结束日期都带下拉箭头。
- 右侧有筛选图标。
- 点击筛选图标进入高级筛选：账户、成员、分类、商家、标签、币种、交易类型。

当前 Finance 只做了 `全部/年/月`，缺少：

- `更多`
- 日期范围行
- 筛选图标
- 自定义日期范围状态

### 4.3 维度 tabs

横向滚动 tabs：

- 趋势
- 大类
- 小类
- 成员
- 账户
- 商家
- 标签
- 更多菜单

要求：

- 选中项为白色或高亮文字，下方橙色短线。
- tab 行可横向滚动。
- 右侧更多菜单用于容纳后续维度。

当前 Finance 已有主要维度，但需要补：

- 右侧菜单图标。
- 与顶部周期筛选联动的日期范围。
- MCP schema 中补齐 `subcategory`、`tag` 和 filters。

### 4.4 大类/小类占比报表

报表卡片：

- 深色模式下为深灰卡片。
- 标题为 `大类支出`、`小类支出`、`成员支出` 等。
- 标题右侧有列表/展开两个图标按钮。

环形图：

- 中心显示：
  - `总支出`
  - 金额，例如 `2621426.66`
- 环形图必须显示多段颜色。
- 环外显示主要分类标签和百分比。
- 标签要有引导线。
- 小占比分类允许合并显示或只在列表中显示。

排行列表：

每行必须包含：

- 左侧分类图标圆。
- 分类名称。
- 金额，右对齐。
- 笔数，例如 `1058笔`。
- 百分比，右下。
- 进度条。
- 分割线。

排行显示：

- 当前 Finance 实现一次显示完整排行列表。
- 不使用默认前三项、`点击展开` 或 `收起` 入口。

### 4.5 趋势报表

趋势页必须支持：

- 按日、月、年聚合。
- 当前筛选条件摘要。
- 折线或柱状趋势图。
- 同一筛选下的明细入口。

趋势维度的返回数据必须包含：

```text
series[].key
series[].label
series[].amountMinor
series[].count
series[].incomeMinor
series[].expenseMinor
```

### 4.6 下钻行为

点击排行行后打开挖财式底部菜单：

- 趋势统计
- 小类报表（仅大类维度出现）
- 账单明细
- 取消

下钻规则：

- 大类 -> 小类报表：传 `category_parent_id`。
- 大类 -> 账单明细：传 `category_parent_id`。
- 小类 -> 账单明细：传 `category_id`。
- 成员 -> 账单明细：传 `member_id`。
- 账户 -> 账单明细：传 `account_id`。
- 商家 -> 账单明细：传 `merchant_id`。
- 标签 -> 账单明细：传 `tag_id`。

二级页面：

- 左上角返回。
- 左缘右滑返回。
- 不显示退出按钮。
- 保留当前筛选上下文。

## 5. 数据和 API 细化

### 5.1 分类主数据

需要扩展：

```text
finance_categories.icon_key
finance_categories.icon_color
finance_categories.is_quick
finance_categories.quick_order
```

导入规则：

- Wacai XLSX 中出现的新分类自动 upsert。
- 如果命中内置分类名称，补齐 icon_key。
- 未知分类使用 `category-generic`。

### 5.2 报表返回

`finance.get_report` 和 `/api/finance/report` 的 breakdown item 需要补：

```json
{
  "key": "cat_home",
  "label": "居家",
  "amountMinor": 75407623,
  "amount": "754076.23",
  "count": 1058,
  "percentageBasisPoints": 2877,
  "iconKey": "home-house",
  "iconColor": "#28c7af",
  "rank": 1
}
```

### 5.3 明细返回

交易列表需要补充展示字段：

```text
categoryName
parentCategoryName
categoryIconKey
categoryIconColor
accountName
memberName
merchantName
tagNames
currency
amount
occurredAt
note
source
```

### 5.4 MCP schema 补齐

`finance.get_report` schema 必须补：

- `dimension`: 增加 `subcategory`、`tag`。
- `filters`: 支持 category/member/account/merchant/tag。
- `start_date`、`end_date` 支持自定义范围。

`finance.list_transactions` schema 必须补：

- `category_id`
- `category_parent_id`
- `account_id`
- `merchant_id`
- `tag_id`
- `currency`

## 6. 当前实现差异清单

必须修正：

- 全局视觉仍有自定义 Finance/Hermes 控制台痕迹，必须改成挖财深色高保真外观。
- 中文显示仍有乱码风险，UI 文案必须统一为 UTF-8 正常中文。
- 分类图标仍有首字占位逻辑，必须改为 `icon_key`。
- 顶部背景目前接近渐变/装饰效果，必须替换为挖财式模糊位图背景。
- 统计页缺少挖财顶部日期范围行和筛选图标。
- 统计页环形图缺少外侧标签和引导线。
- 统计页报表卡片右上角缺少列表/展开图标按钮。
- 底部导航需要按挖财图标、顺序、颜色重做：账户、计划、首页、报表、我的。
- 记账页删除键、乘除键等存在显示不一致风险，必须固定为图标或明确字符。
- 记账页“再记”需要真实保留页面继续录入，而不是简单提交。
- 记账页商家、标签、附件入口目前只是占位，后续要接入选择/输入流程。
- MCP schema 与 service 支持的报表维度/过滤器不完全一致。

可以暂缓但必须保留扩展位：

- 真正的图片背景素材替换。
- 图标完全逐像素贴近挖财。
- 高级筛选完整页。
- 多币种汇总折算。
- 周期记账、预算、计划待办。

## 7. 实施顺序

### Phase A：界面和文案基线

1. 修复 UI 中文乱码。

## 9. 验收标准

记账页验收：

- 视觉上第一屏与挖财截图一致：蓝色模糊顶部、四个类型 tab、深色分类金额卡、4 列分类网格、备注行、chips、深灰键盘、橙色保存。
- 分类图标不是文字占位。
- 金额、日期、账户、成员、标签、商家、相机入口顺序一致。
- 保存、再记、删除、计算键可用。
- 页面不能出现乱码、重叠、横向滚动或底部安全区遮挡。

统计页验收：

- 顶部周期 segmented、日期范围、筛选图标、维度 tabs 与挖财布局一致。
- 大类统计卡片为深色圆角卡片。
- donut 中心显示总支出和金额。
- 分类标签、百分比、排行行、进度条与截图结构一致。
- 底部导航为挖财顺序和选中态。
- 点击排行行后的下钻菜单和二级页返回行为保持可用。

架构验收：

- 视觉复刻不改变 Hermes MCP-only 集成边界。
- 不把挖财账号、密码、导出原始文件或私有资源写入 git。
- 图标和背景进入本项目本地资产目录后再引用。
